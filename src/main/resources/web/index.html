Sorry,

IN-DEV versions are not meant to use the web interface. If you want to use the web interface, you need to start the interface in a separate process.

(Recommended) How to start the LIVE web interface:

  1. Install the latest LTS (currently 16.14.2) version Node.js and NPM on your machine (download).
  2. Install the PNPM with the command "npm install -g pnpm"
  3. Go into the "website" folder in the root directory of the project.
  4. Download and install all dependencies with the command "pnpm i"
  5. Copy the "default.env" file as an ".env" file.
  6. If you are not using the default port for the api (the kotlin/java application) you need to change the port in the ".env" file.
  7. Start the server with the command "pnpm dev"
  8. Open the web interface in your browser: http://localhost:3000

How to build the web interface into the API:

  1. Install the latest LTS (currently 16.14.2) version Node.js and NPM on your machine (download).
  2. Install the PNPM with the command "npm install -g pnpm"
  3. Go into the "website" folder in the root directory of the project.
  4. Download and install all dependencies with the command "pnpm i"
  5. Copy the "live.env" file as an ".env" file.
  6. Build & Generate the static interface files with the command "pnpm build"
  7. The next steps should be done from the root directory of this project.
  8. Copy the files from the "website/dist/" folder into the "src/main/resources/web/" folder and overwrite the "index.html" file (this file) in it.
  9. Delete the "web.files/" folder.
  10. Start the API server which should now display the interface instead of this info.
If you are wondering why the web interface is not by default available, the reason is that the web interface is meant to be freshly build by our CI jobs on every commit & release.
